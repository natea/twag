## Description

Multimodal Hacks: Build the Interface for Agents

On June 6, 2026, join Forever 22, Google DeepMind, Pinecone, LangChain, Cursor, Clerk, and Betaworks for a full-day, hands-on hackathon focused on one idea: the interface layer for AI agents is expanding far beyond the command line.

We’re bringing together engineers, designers, and builders working across agents, LLMs, voice, generative media, developer tools, and real-time experiences to prototype the next generation of agent interfaces.

Whether you’re building voice-first agents, multimodal copilots, creative tools, workflow assistants, or new ways for people to interact with AI, this is a space to build something ambitious, useful, and demo-ready in a single day.

You’ll spend the day building alongside other top builders. Look forward to sponsor activations, an AMA, hacking time, demos, food, and prizes.

We’ll have up to five submission categories — and five winning teams.

Teams of up to 3 people. To keep things fair and fresh, all projects must be started from scratch on the day of the hackathon.

We’re accepting people on a rolling basis.
 
MC: Mari Zumbro(Filament) 

Judges: Marc (Danger Testing), Tommy (ClawCon), Yiliu (Collaborator AI), and Kaya Jones(Forever22 / Betaworks)

AMA Speaker: Vince Trost (CEO, Plastic Labs)

ABOUT OUR SPONSORS:

Google DeepMind is Google’s leading AI research lab, building frontier models like Gemini and pioneering breakthroughs across science, robotics, and generative media. 

Pinecone is a managed vector database that gives AI agents fast, scalable long‑term memory for search, RAG, and recommendations. 

LangChain is an open‑source framework for building AI agents, with tooling and integrations that make it easy to orchestrate models, tools, and data.

Cursor is an AI-native code editor and coding agent that helps developers plan, write, and ship software faster with deep codebase understanding.

Clerk is a modern authentication and user management platform that provides drop‑in sign-in, orgs, and billing-aware user flows so teams can focus on product, not auth.
