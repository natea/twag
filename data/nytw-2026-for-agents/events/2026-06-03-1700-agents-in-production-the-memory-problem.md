## Description

Agent memory is a mess. 

Most teams are stitching together a vector store, a graph database, and some middleware - three systems to manage, three potential failure points, and no guarantees any of it stays consistent. It works until it doesn't.

This meetup is about solving it properly. Not a quick fix, not another abstraction layer - memory that's transactional, structured, and built into the database itself. We'll show what that looks like in practice with Spectron, SurrealDB's brand new agent memory system.

We'll be joined by engineering teams from Pydantic and Tavily - covering how to structure and validate the data your agents work with, and how to handle retrieval at runtime without losing the plot mid-task.

Hard questions welcome. Drinks on us.
