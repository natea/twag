# Stagehopper demo — 15-shot screen recording shotlist

**Goal:** ~15-20s clip showing browsing + searching for the AGI House / Sundai talk on 2026-05-31.

**Tool:** macOS Cmd+Shift+5 → "Record Selected Portion" (or CleanShot X). Record at 1280×800 if possible — projects cleanly to 16:9. Mouse cursor on, no audio.

**URL to open:** https://stagehopper.app (once DNS propagates) — fallback: https://natea.github.io/twag/events_map_nyc.html

## The 15 beats

1. Open https://stagehopper.app — landing on Boston map, pins clustered.
2. Pause 1s on the cluster view so the density reads.
3. Zoom in on Cambridge/Kendall cluster — clusters break apart into individual pins.
4. Click a pin in Kendall Square — sidebar slides in with event card.
5. Hover the event card title (cursor visible).
6. Click back to map.
7. Switch the date picker to **May 28** — pins re-shuffle.
8. Click the **Gallery** tab.
9. Scroll the gallery 2-3 thumb-flicks — posters fly by.
10. Click into the search box.
11. Type **"AI"** — gallery filters live.
12. Clear search, type **"founder"** — filters again.
13. Click a poster tile — opens the event modal/RSVP link (don't click through to Partiful, just show it).
14. Hit back, switch the city toggle from **Boston → NYC**.
15. NYC map loads, June 1 default date, 1,300+ pins cluster across Manhattan/Brooklyn. End on that wide shot.

## Production notes

- If a take goes >25s, redo — 2 min talk only has 15-20s of headroom for video.
- If the live site is mid-DNS-propagation at record time, fall back to https://natea.github.io/twag/events_map_boston.html so the URL bar reads cleanly. Re-record once stagehopper.app resolves with HTTPS.
- Export as .mp4 (H.264). Drop into Google Slides via Insert → Video → Upload from computer (it'll go to Drive).
- No music. The talk is the audio.
